How you play

Game mechanics

One clear loop. Win and lose for real — jokes are free, hearts are not.

Explore → Collect → Redeem → Eat → Survive × 7

Seven fast days

About 60 seconds each. Eat before the timer ends or lose a heart. Last until day 7 — Agentur für Arbeit finally pays.

Mystery bins

Dig for Pfand bottles… or burn your hand. Outcome stays hidden until you commit. Slot-machine energy, Berlin edition.

Bottles → cash → Döner

Queue at REWE for Pfand cash, then buy food at the Döner stand. Waits cost precious seconds while the day clock keeps ticking.

Three hearts

Miss a meal: −1 heart. Bin burn: −½ heart. Hit zero and the run is over. Survive with health left — you win.

How we built it

Technology

Spec first, code second — so four people (and a few agents) could actually run in parallel.

  1. 1
    Record the chaos Whiteboard brainstorm on camera — premise, loop, maps, what to cut. Captured as a transcript so nothing funny got lost.
  2. 2
    Hackathon planning docs Filled the organizers’ ideation / architecture templates so the problem, constraints, and stack were written down before the sprint.
  3. 3
    Transcript → specs Turned the recording into a PDD + technical SPEC, then split workstreams: app infra, game mechanics, assets, sound & dialogue.
  4. 4
    Foundation, then fork Land a shared WP-0 (grid, movement, items, HUD shell), tag it, and parallelize collectibles, economy, run loop, and polish.
